# 데이터 수집 및 분석 방법론

이 프로젝트가 어떤 식으로 뉴스 헤드라인을 수집하고 키워드 빈도를 카운트했는지 정리한 문서.

---

## 1. 핵심 개요

**"하루 100건 받아서 그 중 키워드 있는 것만 분석"이 아닙니다.**

실제 흐름은:
1. **넓은 그물망 키워드 30개**로 Google News RSS를 600번 호출 (날짜 윈도우 포함)
2. 받은 헤드라인을 중복 제거 후 발행일(KST)로 정리
3. **분석용 키워드 14개**가 헤드라인 제목에 나타나는지 1건씩 체크 → 일별 빈도 매트릭스 생성
4. 일별 유가와 시차(lag) 상관분석

---

## 2. 두 종류의 키워드 — 헷갈리지 말 것

| 구분 | 검색용 쿼리 (30개) | 분석용 키워드 (14개) |
|------|--------------------|---------------------|
| 목적 | Google News RSS에서 분쟁 관련 헤드라인 끌어오기 | 끌어온 헤드라인에서 빈도 카운트 |
| 성격 | 넓은 그물망 (broad search) | 좁은 분석 대상 (target tags) |
| 예시 (한국) | "이란 미국", "이란 군사", "헤즈볼라", "솔레이마니" | "호르무즈 해협", "혁명수비대(IRGC)", "테헤란 정권" |
| 코드 위치 | `RSS_QUERIES_KR`, `RSS_QUERIES_US` | `KEYWORDS_KR`, `KEYWORDS_US` |

### 검색용 쿼리 전체 (한국)
```
"이란 미국", "이란 분쟁", "이란 군사", "이란 핵", "이란 제재",
"이란 공습", "이란 미사일", "이란 보복", "이란 공격", "이란 위기",
"호르무즈", "혁명수비대", "IRGC", "헤즈볼라", "JCPOA",
"핵협상", "솔레이마니", "샤헤드", "테헤란", "중동 정세",
"중동 분쟁", "페르시아만", "시아파", "이스라엘 이란", "이란 전쟁",
"트럼프 이란", "후티 이란", "예멘 이란", "이란 드론", "이란 협상"
```

### 분석용 키워드 (사용자 지정)
- 호르무즈 해협, 혁명수비대(IRGC), 시아 벨트, 테헤란 정권
- 샤헤드 드론, 대리세력 네트워크, 핵협상(JCPOA), 중동 저항축
- 헤즈볼라 연계, 페르시아만 긴장, 제재 경제, 비대칭 전력 전략
- 시아파 영향권, 솔레이마니 사망 사건

별칭이 있는 경우(예: 혁명수비대 / IRGC) 헤드라인에 둘 중 하나라도 등장하면 +1 카운트.

## 3. 수집 흐름 (전체 단계)

### Step 1. Google News RSS 호출 — 600 쿼리 (30 키워드 × 20 윈도우)

```
쿼리 #001: "이란 미국 after:2026-01-01 before:2026-01-08"
           → 헤드라인 100건 받음 (발행일은 1/1~1/7 사이로 다양)
쿼리 #002: "이란 미국 after:2026-01-08 before:2026-01-15"  
           → 헤드라인 100건
...
쿼리 #600: "이란 협상 after:2026-05-13 before:2026-05-20"
           → 헤드라인 N건
```

**7일 윈도우의 이유**: Google News RSS는 쿼리당 최대 100건만 반환. 전체 140일을 한 쿼리로 묶으면 최신 100건만 받게 됨. 7일 단위로 쪼개야 1월~5월 골고루 수집됨.

### Step 2. 누적 + 중복 제거

```
누적 raw: ~50,000건 (600 쿼리 × 평균 80건)
   ↓ 제목 기준 중복 제거
한국 21,455건, 미국 28,843건
```

### Step 3. KST 기준 날짜 컬럼 생성

각 헤드라인의 `published` 값(GMT/UTC)을 `Asia/Seoul`로 변환 후 `date` 컬럼 추출.

### Step 4. 분석 키워드 매칭 → 일별 카운트

```
헤드라인 예시: "호르무즈 해협 통항 위협, 미국 항모 급파 - 연합뉴스"
  → 호르무즈 해협 별칭("호르무즈") 있음     → +1
  → 다른 13개 키워드는 없음

헤드라인 예시: "이란 혁명수비대 IRGC 보복 천명 - 조선일보"  
  → 혁명수비대(IRGC) 별칭 둘 다 등장         → +1 (중복 카운트 X)
  → 솔레이마니 사망 사건 별칭 없음
```

### Step 5. 일별 매트릭스로 집계

```
date         호르무즈 해협  혁명수비대(IRGC)  ...  total_headlines
2026-01-01        0              1          ...        16
2026-01-02        0              0          ...        40
2026-01-07        5              3          ...        45
...                                                    
2026-05-20       18              9          ...       198
```

- `total_headlines`: 그날 수집된 전체 헤드라인 수 (정규화 용도)
- 키워드 컬럼: 별칭 매칭으로 카운트된 헤드라인 수

## 4. 흔한 오해 vs 실제

| 가정한 방식 | 실제 방식 |
|------------|----------|
| 하루 100건 데일리로 받음 | RSS는 그런 식으로 못 줌 (쿼리 + 날짜 윈도우 기반) |
| 매일 정확히 100건 보장 | 일별 16~240건으로 변동 |
| 받은 뒤 키워드 있는 것만 본다 | 모든 헤드라인 보관 + 키워드 매칭은 별도 카운트 |
| 키워드 없으면 버림 | 안 버림 → `total_headlines`에 카운트 |
| 7일 윈도우 = 7일을 한 묶음으로 분석 | 분석은 1일 단위, 윈도우는 수집 트릭일 뿐 |

## 5. 7일 윈도우의 역할 (재강조)

```
[수집 단계 — 7일 윈도우 사용]
                                              
"호르무즈" + after:2026-01-01 before:2026-01-08  →  헤드라인 100건
"호르무즈" + after:2026-01-08 before:2026-01-15  →  헤드라인 100건
"호르무즈" + after:2026-01-15 before:2026-01-22  →  헤드라인 100건
...                                         

[저장 단계 — 발행일 그대로 보존]
                                              
title                      | published_kst        | date
"호르무즈 통항 위협..."     | 2026-01-03 08:30 KST | 2026-01-03
"호르무즈 해협 긴장..."     | 2026-01-05 14:00 KST | 2026-01-05

[분석 단계 — 1일 단위 그루핑]
                                              
date         | 호르무즈 해협 | 혁명수비대 | ... | total_headlines
2026-01-01   |      0       |     1     | ... |       16
2026-01-02   |      0       |     0     | ... |       40
(140행, 1일씩)
```

**핵심**: 7일 윈도우는 Google News RSS의 100건 한계를 우회하기 위한 fetch 트릭. 분석은 어디까지나 1일 단위.

## 6. 실제 일별 분포 (v2)

| 통계 | 한국 | 미국 |
|------|------|------|
| 총 헤드라인 | 21,455건 | 28,843건 |
| 고유 날짜 수 | 140일 | 140일 |
| 일별 평균 | 153.2건 | 206.0건 |
| 최소 (가장 적은 날) | 16건 | ~10건 |
| 최대 (가장 많은 날) | ~240건 | ~240건 |

→ 매일 같은 양이 아니라 사건이 많은 날에 헤드라인이 폭증함. 이게 `total_headlines`로 정규화하는 이유.
